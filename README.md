A Continuous Integration and Continuous Deployment (CI/CD) pipeline was implemented using **GitHub Actions**.

### ğŸ”„ How it works

Whenever a developer wants to add a new feature or modify the code:

1. The developer pushes the changes to the GitHub repository.
2. The CI/CD pipeline is automatically triggered.
3. The application is rebuilt and redeployed.
4. The changes are reflected remotely without manual intervention.
